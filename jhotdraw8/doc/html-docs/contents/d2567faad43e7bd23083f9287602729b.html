Description

The idea of the design for undo/redo support in JHotDraw is to implement UndoManagerAdapter(s) that can be connected with Model objects using the Listener support of the model.

An UndoManagerAdapter creates an UndoableEdit for each ModelChangeEvent that it receives from the Model. It adds each created UndoableEdit object to the UndoManager.

Grouping events into user transactions:

A UserInterfaceControl creates a CompoundEdit object and adds it to the UndoManager when the user starts an undoable action. The User Interface Control calls end() on the CompoundEdit when the user has completed the undoable action. The UndoManager aggregates UndoableEdit events into a CompoundEdit until end() has been called on the CompoundEdit.

Note that undo/redo still works if a UserInterfaceControl does not create CompoundEdits however then the user might be surprised that a single action caused multiple UndoableEdits.

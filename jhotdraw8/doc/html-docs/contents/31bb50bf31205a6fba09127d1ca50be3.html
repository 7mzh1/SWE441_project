Description

The Logical View captures the object model of the design.

The logical view shows an object-oriented decomposition of the problem domain (logical architecture).

The logical architecture primarily supports the functional requirements — what the system should provide in terms of services to its users.

The system is decomposed into a set of key abstractions, taken (mostly) from the problem domain, in the form of objects or object classes.
